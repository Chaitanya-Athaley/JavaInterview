Q: What is a record?
A: A record is an immutable data carrier class that automatically generates:
   â€¢ All-args constructor
   â€¢ Getters for all fields
   â€¢ equals() method (compares all fields)
   â€¢ hashCode() method (based on all fields)
   â€¢ toString() method (lists all field values)
   â€¢ They're final classes with final fields

Q: How do sealed classes help pattern matching?
A: Sealed classes limit which types can extend them, allowing the compiler to
   verify all cases are handled. This enables exhaustiveness checking:
   
   switch (shape) {
       case Circle c -> { }
       case Rectangle r -> { }
       case Triangle t -> { }
       // Compiler ensures all cases covered - no default needed
   }

Q: What problem do virtual threads solve?
A: Platform threads are expensive (1-2 MB memory, ~1000-10,000 per JVM).
   Virtual threads are lightweight (1-2 KB, millions possible), allowing:
   â€¢ Web servers to handle millions of concurrent requests
   â€¢ Simpler code (no callbacks or reactive streams)
   â€¢ Automatic context switching at blocking points
   
Q: When would you use sealed records?
A: When you have a fixed set of related, immutable types:
   â€¢ API response types (Success, Error, Redirect)
   â€¢ Result types (Ok<T>, Err<E>)
   â€¢ UI events (Click, Scroll, KeyPress)
   â€¢ State machines (Menu, Playing, Paused, GameOver)
